# Change: 支持主流日志压缩格式

## Why

LogCrate 当前只有 ZIP 读取器。7z、RAR、tar.gz 等文件虽然可能出现在目录树中，但无法展开或读取，用户容易把“文件可见”误认为“格式受支持”。实际日志交付常见于 Windows 的 ZIP/7z/RAR，以及 Linux/macOS 的 TAR 与 gzip/bzip2/xz/zstd 压缩流，需要提供一致的识别、展开、通知和免手动解压阅读体验。

## What Changes

- 建立统一格式注册表，以文件头为主、复合后缀为辅识别归档，避免各模块分别维护后缀列表。
- 在现有 ZIP 与裸文本之外，支持 7z、RAR、TAR、tar.gz/tgz、tar.bz2/tbz2、tar.xz/txz、tar.zst/tzst。
- 支持 `.gz`、`.bz2`、`.xz`、`.zst` 单文件压缩日志，并在目录树中呈现为一个可打开的虚拟日志条目。
- 自动识别归档条目中的受支持嵌套归档，允许用户在目录树中逐层展开并直接阅读最内层日志，无需手动释放任何中间文件。
- 目录监控、拖入检查、递归到达通知、目录树图标和惰性展开统一使用同一格式能力表。
- 保持免手动解压：不向用户目录释放文件，不把完整条目读入内存；顺序格式按需扫描，并由现有索引缓存提供随机阅读能力。
- 对损坏归档、超限解压、加密、分卷和不支持的压缩算法返回可本地化的明确错误。
- 增加各格式的小型夹具、大文件流式读取、伪造扩展名、损坏包、压缩炸弹、取消清理与 Windows/macOS 构建测试。

## Scope

首版支持标准、未加密、单卷归档及其受支持格式的嵌套组合。密码输入和分卷归档（如 `.part1.rar`、`.7z.001`、`.z01`）不在本 change 中；系统必须识别并明确提示。创建、修改、追加、删除条目或重新打包归档明确不在产品范围内。

## Impact

- Affected specs: `archive-reading`、`directory-monitoring`
- Affected code: `src-tauri/src/archive/*`、`src-tauri/src/watcher.rs`、归档类型与错误 API、目录树展示、测试夹具
- Dependencies: 预计新增 TAR 与压缩流解码依赖；7z/RAR 可能引入 libarchive 或格式专用读取器，必须完成许可证、供应链与打包评估
- Performance: 顺序与嵌套归档列目录可能需要扫描或缓存中间条目，但只允许在用户逐层展开时执行；读取与索引必须保持固定缓冲、最大嵌套深度和聚合解码上限
- Packaging: 新依赖不得静默突破安装包体积目标；实现阶段必须记录 Windows/macOS 产物增量并显式处理超限
