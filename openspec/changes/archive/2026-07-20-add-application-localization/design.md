## Context

用户可见文案目前散布于 `App`、目录树、日志正文、顶栏、设置与更新组件、浏览器 mock、Tauri API 适配层和 Rust 托盘菜单中。部分错误来自 Rust 或操作系统，包含动态文件名和路径；日志正文、文件名、Release Notes 等用户数据则不应被翻译。

首批只交付简体中文和英文，但结构必须允许后续通过增加字典扩展日语、韩语等语言，而不再重构组件。

## Goals / Non-Goals

- Goals:
  - 应用固定界面文案完整覆盖简体中文与英文。
  - 首次启动合理跟随系统，用户选择可持久化且即时生效。
  - 前端界面、文件选择器、可访问性标签和原生托盘保持同一语言。
  - 字典缺键在开发和测试阶段可被发现，运行时安全回退英文。
- Non-Goals:
  - 首批不提供繁体中文、日语、韩语、法语等其它翻译。
  - 不翻译日志正文、文件名、磁盘路径、压缩包条目、版本更新说明或底层操作系统原始错误文本。
  - 不引入 ICU MessageFormat、在线翻译服务或第三方 i18n 框架。

## Decisions

### 使用强类型扁平消息字典

- 新增 `src/i18n`，英文作为完整基准字典，简体中文字典必须满足同一键集合。
- 消息键采用语义化命名，例如 `settings.language`、`tree.removeWatch`、`update.download`，组件不得再内嵌用户可见固定文案。
- `t(key, params)` 支持简单命名插值；数字和字节等已有格式化逻辑继续复用，不为两个首批语言引入复杂复数规则。
- 运行时缺键回退英文并在开发环境输出警告，测试比较两套字典键集合。

### 区分语言偏好与实际语言

- 持久化值为 `system | zh-CN | en`；`system` 每次启动根据 `navigator.languages` 解析实际语言。
- 任一 `zh-*` 系统语言首批映射到 `zh-CN`，其它未支持语言映射到英文。
- 用户切换后同步更新 React Context、`document.documentElement.lang` 和本地存储；现有打开文件、选项卡与查看状态不得重置。
- 存储不可用或值损坏时安全回退 `system`。

### React Context 负责组件消费

- 应用根部提供 `I18nProvider`，暴露实际 locale、偏好、`t` 和变更函数。
- 高频日志行不调用翻译函数处理正文；仅加载中、截断等固定状态读取字典，语言切换不会重新解析日志内容。
- 非 React 工具函数需要用户文案时接收翻译函数或返回稳定错误代码，避免直接依赖全局单例。

### 托盘文案由前端字典同步

- Rust 在启动时先使用英文安全默认值创建托盘项，并管理菜单项句柄。
- 新增受限 Tauri 命令接收受支持 locale 或两项已知标签，调用 `MenuItem::set_text` 即时更新“显示主窗口”和“退出 LogPeek”。
- 前端启动及每次实际语言变化时同步一次；失败不阻断主界面切换。

### 错误与外部内容边界

- 前端产生的错误、确认与操作状态全部使用消息键。
- 后端已知业务错误通过稳定错误代码或归一化映射转换为当前语言；动态文件名和路径作为插值参数保留。
- 无法分类的 Rust、网络和操作系统错误保留原始详情，但外层动作前缀（如“打开失败 / Open failed”）必须翻译。
- GitHub Release Notes、日志正文和磁盘内容始终原样展示。

## Risks / Trade-offs

- 文案面广，容易遗漏 → 全文扫描用户可见中英文硬编码，并用字典键一致性测试与双语人工走查兜底。
- 语言切换导致大组件重渲染 → Context 只改变界面文案；日志行正文 memo 化且不做翻译解析。
- 托盘创建早于 WebView 初始化 → 启动先显示英文默认值，前端加载后立即同步实际语言。
- 原始底层错误无法完全翻译 → 翻译操作前缀和已知业务错误，保留原始详情用于诊断。

## Migration Plan

现有用户首次升级时没有语言偏好，将进入“跟随系统”。删除本地化 Provider、语言设置和托盘同步命令即可回滚；监控目录、筛选、选项卡和更新设置数据格式不变。
