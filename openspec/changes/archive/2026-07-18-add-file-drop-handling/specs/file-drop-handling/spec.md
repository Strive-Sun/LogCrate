## ADDED Requirements

### Requirement: 本地文件拖入检查
系统 SHALL 通过桌面 WebView 原生文件拖放事件接收本地绝对路径，并在执行监控或打开操作前由后端规范化路径、确认普通文件或目录可读并识别路径类型。本 change SHALL 一次只处理一个路径；多路径、失效路径和不支持的特殊文件类型 MUST 整体拒绝且不得产生部分操作，多路径支持留给后续 change。

#### Scenario: 拖入单个普通文件
- **WHEN** 用户从系统文件管理器拖入一个可读取的任意普通文件
- **THEN** 系统完成后端路径和类型检查，以文件父目录作为实际监控路径，并仅在文件为可识别裸文本日志时标记为自动打开

#### Scenario: 拖入多个路径
- **WHEN** 一次 drop 事件包含多个路径
- **THEN** 系统提示一次仅支持一个文件或文件夹，且不添加任何监控目录或打开其中任意文件

#### Scenario: 拖入目录
- **WHEN** 用户拖入一个可读取目录
- **THEN** 系统接受该路径并将目录自身作为实际监控路径，不执行日志打开

#### Scenario: 拖入不支持或失效路径
- **WHEN** 路径不存在、不可读取或既不是普通文件也不是目录
- **THEN** 系统显示可读错误，且不修改监控配置或当前查看会话

#### Scenario: 拖入悬停不遮挡界面
- **WHEN** 支持桌面拖放的路径进入、悬停或离开主窗口
- **THEN** 系统不显示覆盖窗口的遮罩、弹窗或处理中提示，且后续 drop 仍可正常接收

### Requirement: 拖入路径自动监控
系统 SHALL 在拖入路径检查成功后自动添加监控：拖入普通文件时添加其父目录，拖入文件夹时添加文件夹自身作为监控根，不显示加入监控确认弹窗，且不得移动、复制、重命名、上传或删除原路径。目标已被现有根覆盖时 SHALL 跳过重复添加；新增根覆盖已有子级根时 SHALL 归并子级配置和 watcher，确保任一路径不会被父子根重复监控。

#### Scenario: 自动监控任意文件所在目录
- **WHEN** 用户拖入任意普通文件且其父目录尚未被监控
- **THEN** 系统将父目录添加为监控根，且不移动、打开或修改非日志文件

#### Scenario: 自动监控拖入的文件夹
- **WHEN** 用户拖入文件夹且该目录尚未被监控
- **THEN** 系统将文件夹自身添加为监控根并刷新目录树

#### Scenario: 自动监控 zip 所在目录
- **WHEN** 用户拖入 zip 且其父目录尚未被监控
- **THEN** 系统将父目录添加为监控根、刷新目录树并定位该 zip，原文件位置和内容不变

#### Scenario: 父目录已被监控根覆盖
- **WHEN** 拖入文件已位于某个现有监控根的任意深度子目录
- **THEN** 系统不增加嵌套监控根，并在目录树中逐层定位 zip 或裸日志，同时继续打开裸日志正文

#### Scenario: 新监控根覆盖已有子级根
- **WHEN** 用户确认添加的目录是一个或多个现有监控根的祖先目录
- **THEN** 系统添加该祖先根并移除被覆盖的子级根配置与 watcher，使覆盖范围内只保留一个有效监控根

#### Scenario: 删除确认弹窗显示期间拖入
- **WHEN** 删除确认或更新弹窗仍在显示时用户 drop 路径
- **THEN** 系统不替换现有弹窗、不处理新路径，并提示先完成当前操作

### Requirement: 拖入裸日志自动打开
系统 SHALL 在后端确认拖入文件为 `.log`、`.txt`、`.json` 或其它可识别裸文本日志后立即使用现有查看会话打开并显示内容。监控添加失败或目录已受监控均不得关闭已经成功打开的日志。

#### Scenario: 拖入日志并自动监控
- **WHEN** 用户拖入 `.log`、`.txt` 或其它可识别裸文本日志
- **THEN** 系统立即开始打开日志，同时添加并刷新监控目录、定位并选中该日志

#### Scenario: 拖入已受监控的日志
- **WHEN** 裸文本日志已经位于现有监控根覆盖范围内
- **THEN** 系统不显示监控确认，立即打开日志内容，并逐层展开目录树、选中文件并滚动到可见区域

#### Scenario: 当前日志不匹配后缀筛选
- **WHEN** 拖入文件通过内容采样识别为文本日志，但其扩展名不匹配当前目录后缀筛选
- **THEN** 系统仍在目录树中显示并定位当前查看文件，定位完成后清理自动展开状态且不影响其它被筛选文件

#### Scenario: 拖入大日志
- **WHEN** 拖入需要后台索引的大型裸文本日志
- **THEN** 系统立即创建查看会话并使用现有索引进度反馈，拖放处理和 UI 不等待完整索引
