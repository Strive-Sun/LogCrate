## Context

Tauri 2 的 `getCurrentWebview().onDragDropEvent` 会发送 `enter`、`over`、`drop`、`leave` 事件，其中 `drop` 在桌面端携带本地绝对路径。浏览器原生 `DataTransfer` 在 WebView 中无法可靠提供可用于 Rust 文件 IO 的绝对路径，因此拖放入口应使用 Tauri API。

现有 `add_watch_dir` 已负责持久化并建立 watcher，`open_log_session` 也能直接打开绝对路径形式的裸文件；缺少的是安全、统一的拖入路径检查，以及不经过文件夹选择器添加文件父目录或文件夹自身的前端 API。

## Goals / Non-Goals

- Goals:
  - 拖入单个任意普通文件时自动监控其父目录，拖入文件夹时自动监控文件夹自身，不显示加入监控确认弹窗。
  - 拖入单个可识别裸文本日志时自动打开；已受监控或添加成功后在目录树中定位文件。
  - 已受现有监控根覆盖的路径不产生重复配置。
  - 添加父级根时归并已存在的子级根，配置和底层 watcher 不重复覆盖同一目录树。
  - 拖放悬停和处理中不遮挡当前内容，重复事件不会并发覆盖弹窗或查看状态。
  - 路径类型、实际监控路径和日志属性由 Rust 后端校验。
- Non-Goals:
  - 不把文件移动或复制到其它监控目录。
  - 不支持一次拖入多个文件或文件夹；多路径拖入由后续独立 change 设计。
  - 不自动选择 zip 内具体条目或直接打开 zip 内第一份日志。
  - 不改变系统外部拖出文件、剪贴板粘贴或命令行打开行为。

## Decisions

### 使用 Tauri 原生拖放事件

- `App` 挂载时订阅 `getCurrentWebview().onDragDropEvent`，卸载时解除监听。
- `enter`、`over` 和 `leave` 不改变 UI；仅 `drop` 携带的路径进入串行处理。
- 使用处理中 guard 防止快速重复 drop 覆盖现有检查、监控添加或打开任务；已有删除确认或更新弹窗时拒绝新 drop 并给出明确提示。
- 浏览器 mock 不注册原生文件事件；单元测试覆盖事件到动作的纯逻辑。

### 后端统一检查拖入路径

- 新增只读命令 `inspect_dropped_file(path)`：规范化路径，确认它是可读取普通文件或目录，识别为 `directory`、`archive` 或 `file`，并返回规范化路径、显示名、`watch_path`、`is_log` 及 `already_monitored`。
- 任意可读普通文件均可触发父目录监控；zip 与裸文本判定复用监控到达检测的分类能力。显式拖入不受当前通知后缀筛选影响，例如可识别的 `.json` 文本仍可直接打开。
- `already_monitored` 按规范化后的路径包含关系判断；Windows 忽略路径组件大小写，macOS 遵循当前文件系统规范化结果。
- 已删除、权限不足、非普通文件且非目录或文件父目录无法确定时返回可读错误，前端不继续添加监控或打开。

### 拖入后自动添加实际监控路径

- 普通文件和归档的 `watch_path` 是父目录，文件夹的 `watch_path` 是文件夹自身。
- 后端检查成功后立即调用不弹选择器的 `addWatchPath(watchPath)`，复用现有 `add_watch_dir` 命令完成路径规范化、覆盖去重、持久化、递归到达 watcher 与根目录结构 watcher 初始化。
- 目标已受其它监控根覆盖时跳过重复添加；新增根覆盖现有子级根时删除冗余子级配置和 watcher。拖放不移动、复制或修改原路径。
- 添加成功或文件已受监控时，zip 与裸日志均在现有目录树定位逻辑中逐层显示、选中并滚动到可见区域；定位失败不回滚已经成功的监控配置，也不关闭已打开的裸日志。

### 裸日志打开与监控添加互不依赖

- 后端确认拖入项为裸文本日志后立即启动现有 `openLogSession`，同时自动添加实际监控路径。
- 添加监控失败或父目录已被覆盖，都不关闭已经成功打开的裸日志会话。
- 父目录已受监控或添加成功后，目录定位只复用已启动的查看会话，不重复打开裸日志。
- 当前查看的日志即使不匹配后缀筛选也在目录树中保持可见，使定位目标能够挂载、完成滚动并及时清理目录展开链。
- 日志打开失败使用现有错误处理并清理打开中状态；监控添加仍可独立完成。

### 单文件和弹窗队列边界

- `paths.length !== 1` 时整体拒绝，不选择“第一个”造成隐式数据丢失。
- 单个可读普通文件或目录均接受；其它特殊路径整体拒绝。
- 现有删除确认和更新弹窗具有更高优先级；其显示期间拒绝处理新拖入路径。

## Risks / Trade-offs

- 自动打开大日志会立即启动索引 → 复用现有后台索引、进度和取消机制，UI 不阻塞。
- 文本内容采样可能把未知扩展名识别为日志 → 后端复用现有分类规则；拖入是用户明确操作，仍允许直接查看。
- 没有悬停提示会降低可发现性 → 拖放是辅助快捷入口，保留现有“添加监控目录”按钮作为明确入口。
- 拖入会直接持久化监控根 → 全窗口提示明确说明拖入行为，监控可通过现有“移除监控”操作撤销，原路径始终不被修改。
- 父目录可能已被更高层根覆盖 → 后端返回覆盖状态并跳过重复添加。

## Migration Plan

无需配置格式迁移。升级启动时会按规范化覆盖关系归并已有父子监控根；由于父级根完整覆盖被移除的子级根，监控范围不会缩小。回滚仅移除拖放入口，查看会话格式不受影响。

## Resolved Questions

- 用户已确认拖入文件使用父目录、拖入文件夹使用自身作为监控根，不复制或移动原路径。
- 用户已确认任意普通文件均可触发目录监控，只有可识别文本日志自动打开。
- 用户已确认拖入后直接添加监控，不显示加入监控确认弹窗。
- 用户已确认本 change 只实现单路径拖入，多路径拖入延后到下一项 change。
