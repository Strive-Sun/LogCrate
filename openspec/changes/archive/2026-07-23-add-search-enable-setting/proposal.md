# Change: 增加搜索功能启用设置

## Why

全局文件搜索会在后台建立和维护本地索引，但并非所有用户都需要该能力。当前应用启动后会直接恢复搜索索引，用户无法在设置中关闭这部分后台工作，也无法在首次使用前主动选择是否启用。

## What Changes

- 在设置面板增加“启用文件搜索”开关，新安装和缺少有效配置时默认关闭。
- 搜索开关只保存下一次启动的偏好；每次开启或关闭后都提示“将在下次启动时生效”，当前进程不动态启动或销毁搜索后端。
- 应用启动时读取持久化偏好；搜索关闭时不恢复索引、不启动文件系统增量监听，也拒绝搜索命令。
- 用户在本次运行搜索未启用时无法点击顶栏搜索入口，鼠标悬停时提示在设置中开启搜索本地文件功能；搜索快捷键不执行任何操作。
- 搜索已启用时，顶栏搜索入口和快捷键在监控目录页与搜索页之间双向切换，再次触发即可返回监控目录。
- 搜索已在本次运行启用但用户选择下次关闭时，本次运行继续可用，重启后停止。

## Impact

- Affected specs: `application-settings`, `file-search`
- Affected code: Rust 搜索初始化与命令、搜索偏好持久化、前端 API、设置面板、顶栏搜索入口、本地化与测试
- Persistence: 新增独立的版本化搜索功能偏好，不清除已有索引；重新启用后可复用仍然有效的本地索引
