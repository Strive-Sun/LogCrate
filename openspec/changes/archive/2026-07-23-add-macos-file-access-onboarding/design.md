## Context

当前监控配置只持久化目录路径。路径能告诉 LogCrate“上次监控了哪里”，却不能代表 macOS 已授予本次进程访问该位置的能力。重新启动、应用升级或目录位置变化后，恢复路径可能再次遭遇 TCC、App Sandbox 或系统文件选择授权。

macOS 没有供普通应用调用的“申请完全磁盘访问权限”API，也不允许应用自行打开开关。应用能够做的是解释用途、在用户操作后打开对应系统设置页面，并根据实际文件操作结果处理授权不足。另一方面，用户通过系统选择器明确选择的监控根可以用 security-scoped bookmark 持久化授权意图。

## Goals / Non-Goals

- Goals:
  - 首次运行只进行一次清晰、可跳过的完全磁盘访问权限引导。
  - 用户点击操作后尽可能直接打开完全磁盘访问权限设置页。
  - 已选择的监控根跨重启恢复访问，不再为同一路径重复选择。
  - 权限被撤销或书签失效时局部降级，并提供明确恢复入口。
  - 搜索、监控和打开日志共享同一授权状态，不各自重复弹窗。
  - 保持 Windows 行为和现有用户配置兼容。
- Non-Goals:
  - 自动将 LogCrate 添加到完全磁盘访问权限列表或自动开启开关。
  - 绕过 macOS TCC、POSIX ACL、文件保险箱、网络卷凭据或企业策略。
  - 承诺一次普通目录选择即可访问未来从未选择过的任意目录。
  - 读取文件内容来猜测权限是否开启，或向服务器上传权限/路径信息。

## Decisions

- Decision: 首次启动展示 LogCrate 自己的说明弹窗，提供“前往授权”和“稍后”操作。只有用户点击“前往授权”后才调用系统设置深链，避免无用户意图地抢占系统设置窗口。引导完成/稍后状态采用版本化本地标记；该标记只控制是否自动展示，不代表权限已授予。
- Decision: 首选打开 `x-apple.systempreferences:com.apple.preference.security?Privacy_AllFiles`。由于该 URL 不是跨版本稳定的公开授权 API，调用失败或未到达目标 pane 时降级打开“隐私与安全性”主页面，同时在 LogCrate 内显示完整的手动导航路径。原生打开操作通过 macOS `NSWorkspace` 等平台 API 封装，前端不直接执行 shell 命令。
- Decision: 不提供虚假的布尔型“完全磁盘访问权限已开启”结论。macOS 没有可靠公开查询 API；界面使用“未检查 / 当前操作可访问 / 访问受限”语义，依据用户触发的真实目录枚举或文件打开结果更新。失败分类必须区分权限拒绝、路径不存在、介质离线和普通 I/O 错误。
- Decision: 通过系统目录选择器添加监控根后，在 macOS 原生侧创建 security-scoped bookmark。配置保存规范化路径、书签格式版本和 opaque bookmark bytes；前端只接收目录标识与可展示路径，不接收书签内容。
- Decision: 应用启动恢复监控前解析书签，处理 stale bookmark，并为成功恢复的根调用 `startAccessingSecurityScopedResource()`。访问句柄由后端按规范化根路径引用计数，覆盖 watcher、目录树、搜索结果打开和日志读取；移除最后一个使用者或进程退出时配对调用 `stopAccessingSecurityScopedResource()`。
- Decision: 旧配置仅含路径时执行兼容迁移。若当前进程仍可读取该路径，先继续工作并把状态标为“未持久授权”；需要持久访问时引导用户通过系统选择器重新确认。不得在没有用户选择的情况下伪造书签，也不得删除旧监控根。
- Decision: stale bookmark 若能安全解析到用户原先选择的资源，则在原生侧刷新并原子替换；无法解析、资源身份不符或授权撤销时，将单个根标记为“需要重新授权”。重新授权必须要求用户选择同一目录或明确确认新的目录，成功前保留原配置以便恢复。
- Decision: Full Disk Access 是可选增强而非应用启动前置条件。未获得它时，监控与搜索继续处理 bookmarks 授权范围和其它当前可读路径；访问受保护目录失败时聚合诊断并提供设置入口，不为每个文件、每次启动循环打开系统设置。
- Decision: 权限错误使用稳定的后端错误码传递，UI 根据操作上下文显示“重新授权目录”或“打开完全磁盘访问权限设置”。日志只记录错误类别和匿名目录标识，默认不记录 bookmark bytes；既有诊断若必须记录路径，仍遵循本地日志策略并避免输出受保护文件内容。
- Decision: 本轮稳定性验收限定为同一次安装、同一个应用构建且 macOS 未撤销权限。完成首次授权后，后续退出、重新启动、恢复监控和访问已授权范围不得再次显示 LogCrate 引导或目录选择器。重新构建、覆盖升级、重新安装或应用身份变化后的 TCC 继承不在本 change 的验收范围，界面也不主动提示该风险。
- Decision: 当前没有 Apple Developer ID，签名和公证不作为本轮实施门槛。继续保留现有 `com.logpeek.app` bundle identifier，不在本 change 中主动改变应用身份；未来取得 Developer ID 后，跨版本权限继承作为单独 change 验证。
- Decision: 实施前仍需确认当前产物是否启用 App Sandbox。若启用，entitlements 至少按实际读写需求配置 user-selected file access 与 app-scoped bookmarks；若未启用，则通过同一书签抽象和实机测试验证当前构建的行为，不能混用两种模型的假设。

Alternatives considered:

- 仅保存路径：实现最简单，但无法保存用户通过系统选择器表达的授权，不能解决重启后重复授权。
- 首次启动直接强制打开系统设置：减少一次点击，但会抢占用户操作且仍无法完成授权；采用解释弹窗后的显式操作。
- 修改 TCC 数据库、脚本化系统设置或要求关闭 SIP：不受 Apple 支持且破坏安全边界，明确拒绝。
- 只要求完全磁盘访问权限：无法自动授予，也不应成为监控一个普通目录的必要条件；采用 bookmark 默认路径和 Full Disk Access 可选增强。
- 仅依靠一次权限探测：不同系统版本、隐私域和企业策略下会产生误判；改为基于具体操作的状态与错误分类。

## Risks / Trade-offs

- 系统设置深链可能随 macOS 版本变化 → 提供主页面降级、手动路径文本和目标 macOS 版本实机矩阵。
- 用户在设置中开启权限后，当前进程可能需要重启 → 返回应用后先重试原操作；仍受限时明确提示退出并重新打开，不宣称即时生效。
- bookmark 可能 stale、损坏或指向已移动资源 → 校验解析结果与资源身份，局部重新授权，不静默改监控目标。
- 同一根被 watcher、搜索和查看会话同时使用时容易过早释放 → 后端集中管理引用计数，并为重复恢复、移除与退出增加测试。
- 完全磁盘访问权限扩大可读取范围 → 首次引导说明用途，坚持本地处理、最少日志和用户可撤销；不因获得权限默认加入网络或可移动卷。
- 未签名构建在重新编译、覆盖升级或重新安装后可能被 TCC 视为不同应用 → 明确排除在本轮验收范围之外，未来取得 Developer ID 后单独解决；当前用户界面不展示该升级风险。
- 不同分发渠道的沙箱模型不同 → 本 change 首次发布只支持一个经过验证的 macOS 分发模型，其他渠道另行验证后启用。

## Migration Plan

1. 增加向后兼容的权限配置版本和 bookmark 存储；读取旧的纯路径配置但不删除或重排监控根。
2. 为 macOS 增加原生系统设置打开、bookmark 创建/解析和访问句柄管理；Windows 提供无行为变化的条件编译路径。
3. 升级用户在新版本首次启动时看到一次说明；选择“稍后”不阻止现有可读目录启动。
4. 用户下一次通过选择器确认监控根时为该根补齐 bookmark；失败根可从目录状态菜单单独重新授权。
5. 使用同一个安装构建在支持范围内的 macOS 版本执行首次授权、多次重启、撤销权限和目录移动验证；确认失败时可回滚到旧配置读取逻辑且不丢失路径列表。
