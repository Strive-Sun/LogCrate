# Change: 新增多文件查看选项卡与折叠栏

## Why

当前正文区一次只保存一个查看目标，打开新文件会直接替换旧文件，用户在多个日志之间对比时必须反复回到目录树重新定位和打开。需要把已打开文件保留为选项卡，并在横向空间不足时用折叠栏管理溢出的选项卡。

折叠栏不是简单的只读列表：用户从折叠栏选择文件后，该文件必须进入可见区域并成为当前文件，同时把原来最右侧的可见选项卡交换到折叠栏，保证行为稳定且可预测。

## What Changes

- 将正文顶部单文件标题改为多文件选项卡栏；每个唯一文件或压缩包条目对应一个选项卡，重复打开只激活已有选项卡。
- 根据正文区域可用宽度动态计算可见选项卡数量，超出容量的选项卡进入“更多”折叠栏，当前活动选项卡始终保持可见。
- 点击折叠栏中的选项卡时，将其激活并放入可见区域，替换当前最右侧的可见选项卡；被替换项进入该折叠项原来的位置。
- 可见区域已满时打开新文件，采用相同的最右侧替换规则；窗口扩缩时自动在可见区和折叠栏之间迁移选项卡。
- 支持关闭可见或折叠选项卡；关闭活动项后按邻近顺序选择下一项，关闭会释放对应后端查看会话和临时缓存。
- 切换选项卡保留每个文件的编码、索引进度、滚动位置和已加载行缓存；后端资源超过并发上限时允许旧会话休眠，重新激活时透明重建。
- 增加选项卡分配、折叠交换、窗口缩放、重复打开、关闭选择和会话回收测试。

## Impact

- Affected specs: `log-viewing`
- Affected code: `src/App.tsx`、`src/components/LogContent.tsx`、新增选项卡组件与状态工具、`src/api/tauri.ts`、`src/api/mock.ts`、`src-tauri/src/index.rs`、`src-tauri/src/lib.rs`
- Data: 选项卡仅存在于当前应用进程，不新增持久化配置；重启后不恢复选项卡
- Resources: 前端保留轻量选项卡状态，后端继续使用有界活动会话与 LRU 回收，避免大日志多开导致资源无上限增长
- Dependencies: 复用 React、CSS 与现有虚拟滚动能力，不新增 npm 或 Cargo 依赖
