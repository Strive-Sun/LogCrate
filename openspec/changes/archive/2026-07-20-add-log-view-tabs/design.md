## Context

当前 `App` 仅保存一组 `activeKey` 与 `session`，`LogContent` 也只有一份索引进度、编码、滚动和行缓存状态。`SessionManager` 最多保留 5 个后端会话，超过后通过 LRU 回收，但前端目前没有“选项卡仍存在、会话已经休眠”的表达。

多文件选项卡需要同时解决三个问题：稳定的可见/折叠分配规则、每个文件独立的查看状态、以及有界后端资源。仅把标题渲染成多个按钮会让切换丢失状态，也会在打开大量大文件时累积缓存。

## Goals / Non-Goals

- Goals:
  - 打开多个唯一日志时创建并保留独立选项卡。
  - 宽度不足时把溢出项放入折叠栏，并严格实现“点击折叠项替换最右侧可见项”。
  - 活动选项卡始终可见，重复打开不创建副本。
  - 切换时保留滚动、编码、索引进度和已加载行状态。
  - 显式关闭立即释放资源，超出后端上限时透明休眠与恢复。
- Non-Goals:
  - 不支持拖拽调整选项卡顺序、固定选项卡或多行选项卡栏。
  - 不在应用重启后恢复选项卡或滚动位置。
  - 不支持分屏同时显示多个日志正文。
  - 不改变本 change 之外的多文件拖入限制。

## Decisions

### 使用稳定的选项卡标识

- 使用现有 `entryKey` 作为选项卡唯一键：裸文件为规范化绝对路径，压缩包条目为 `归档绝对路径::包内路径`。
- 打开已存在的键时不创建重复项，只激活该项；如果它位于折叠栏，则执行折叠交换规则。
- 选项卡标题使用文件名或包内条目名，原有绝对路径悬停提示继续保留，用于区分同名文件。

### 显式维护可见区与折叠栏顺序

- 前端维护 `visibleTabIds` 与 `overflowTabIds` 两个有序集合，而不是每次简单截取总列表，确保交换语义不会被重新排序破坏。
- “最后一个选项卡”定义为当前选项卡栏最右侧的可见项。
- 点击折叠栏第 `i` 项时：该项进入最右侧可见槽位并成为活动项；原最右侧可见项写回折叠栏第 `i` 个位置。
- 可见区已满时打开新文件：新文件进入最右侧可见槽位并激活，原最右侧项追加到折叠栏。
- 若可见区尚有空位，新文件直接追加到可见区。

### 折叠栏布局位置

折叠入口位于右侧正文区域顶部并贴住正文区右边缘。出现折叠项时，可见选项卡均匀吸收剩余宽度，使最后一个可见选项卡紧邻“更多”按钮，两侧均不保留弹性空白。折叠栏以浮层形式从“更多”按钮下方向左展开，覆盖正文顶部区域但不推动正文布局；点击浮层外部或按 `Esc` 关闭。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ LogPeek 顶栏                                                     通知  设置 │
├──────────────────────┬───────────────────────────────────────────────────────┤
│ 监控目录             │ 选项卡栏                                              │

│ ▾ downloads          │ │ app.log│ │ api.log│ │ web.log│ │ 更多 (3)  ▾ │     │

│   ▸ day-02            │                                  ┌────────────────┐   │
│                      │                                  │ worker.log   × │   │
│                      │                                  │ system.log   × │   │
│                      │                                  │ debug.log    × │   │
│                      │                                  └────────────────┘   │
│                      ├───────────────────────────────────────────────────────┤
```

- 可见选项卡从左向右排列；存在折叠项时，可见项在不低于容量计算宽度的前提下均匀扩展，“更多”按钮紧邻最后一项并贴住正文区右边缘；没有折叠项时隐藏该按钮。
- 下拉浮层右边缘与“更多”按钮右边缘对齐，向左展开，避免超出正文区域右边界。
- 折叠项过多时浮层设置最大高度并在浮层内部纵向滚动，不增加第二行选项卡。
- 浮层中的每一项展示文件名、状态和关闭按钮；悬停显示完整绝对路径。

点击折叠项时的交换位置如下：

```text
点击前：可见 [app.log] [api.log] [web.log]    折叠 [worker.log] [system.log]
                                      ↑ 最右侧

点击折叠栏中的 system.log

点击后：可见 [app.log] [api.log] [system.log] 折叠 [worker.log] [web.log]
                                      ↑ 活动项                  ↑ 回到原位置
```

### 根据宽度动态调整容量

- 选项卡栏使用 `ResizeObserver` 监听可用宽度，预留折叠按钮和关闭按钮空间后计算容量，容量至少为 1。
- 缩小时优先把最右侧的非活动项移入折叠栏；活动项不得因自动布局进入折叠栏。
- 放大时按折叠栏顺序把项目补回可见区，直至容量用满。
- 自动缩放只调整分区，不改变活动文件和正文状态。

### 每个选项卡持有独立前端查看状态

- `App` 管理选项卡元数据、活动键与会话状态；`LogContent` 按活动键读取对应状态。
- 每个选项卡至少保存：`OpenSessionResult`、打开/失败/休眠状态、编码状态、索引进度、滚动位置与有限行缓存。
- 切换前记录当前滚动位置，切回后在虚拟滚动容器完成布局时恢复；索引事件按 `sessionId` 路由到所属选项卡，后台选项卡完成索引不得覆盖当前正文。
- 行缓存继续按页加载并设置单选项卡上限；休眠或关闭时清理缓存，避免折叠栏项目无限占用前端内存。

### 保留后端会话上限并支持休眠恢复

- 后端继续维持有界活动会话，不因选项卡数量取消 `MAX_SESSIONS` 保护。
- 打开新会话触发 LRU 回收时，后端向前端返回或发送被回收的 `sessionId`；前端将对应选项卡标记为 `dormant`，但不删除选项卡。
- 激活休眠选项卡时重新调用现有打开流程并重建索引；标题、可见/折叠位置和用户选择的编码保留，正文显示重新加载进度。
- 当前活动会话通过读取和切换触摸 LRU，正常情况下不会被新会话回收。

### 关闭与后继选择

- 每个可见选项卡和折叠项均提供关闭操作，关闭时调用 `close_log_session`；重复关闭或会话已被 LRU 回收时按幂等成功处理。
- 关闭非活动项不改变当前正文。
- 关闭活动可见项时优先激活其右侧邻项，其次左侧邻项；若可见区已空但折叠栏仍有项目，则提升折叠栏第一项并激活。
- 关闭可见项形成空位时，按折叠栏顺序提升第一项填补，但除活动项被关闭外不主动切换正文。
- 删除或重命名磁盘文件时，沿用现有失效处理并关闭指向旧路径的全部相关选项卡。

## Risks / Trade-offs

- 多选项卡缓存增加前端内存 → 对单项行缓存设上限，休眠/关闭时释放，后端活动会话继续受 LRU 限制。
- LRU 回收后切回需要重新索引大日志 → 明确显示重新加载进度，换取资源占用有界。
- 动态宽度计算可能产生抖动 → 预留固定折叠按钮宽度、使用最小选项卡宽度并批量提交一次布局结果。
- 同名文件难区分 → 标题保持简洁，悬停显示绝对路径，折叠栏可展示更长路径摘要。
- 后台索引事件并发到达 → 以 `sessionId` 精确路由，失效/被替换会话的迟到事件必须忽略。

## Migration Plan

无需数据迁移。升级后首次打开日志创建第一个选项卡；回滚后仅恢复单文件正文，不影响监控配置和磁盘日志。选项卡不持久化，因此版本切换不存在会话格式迁移。

## Resolved Questions

- “最后一个选项卡”定义为最右侧可见选项卡。
- 折叠栏采用单个“更多”按钮打开的下拉列表，不使用第二行选项卡。
- 从折叠栏选择项目采用原位交换：被替换的最右侧项写回所选折叠项原位置。
- 新打开文件必须立即可见；容量已满时同样替换最右侧可见项。
- 选项卡支持显式关闭，但不跨应用重启恢复。
