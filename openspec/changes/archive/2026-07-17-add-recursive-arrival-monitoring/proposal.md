# Change: 新增监控根目录递归到达检测

## Why

当前 watcher 只监听监控根目录和用户已展开目录的直接子项。用户监控 `D:\` 后，如果压缩包下载到未展开的深层子目录，应用不会收到文件事件，也就不会执行稳定检测和新日志提示。这与“监控整个目录”的用户预期不符。

直接把现有目录树 watcher 改成递归模式会把深层结构事件错误地送入懒加载目录树，并可能在整盘事件溢出时触发昂贵的全盘重扫。因此需要把“任意深度发现新日志”和“已加载目录结构同步”拆成独立监听职责。

## What Changes

- 为每个有效监控根目录建立递归的文件到达 watcher，使任意深度新建、写入完成或重命名进入的 zip/日志文件都进入稳定检测和通知流程。
- 保持目录树结构 watcher 非递归且按展开状态创建/释放；递归到达事件不提前填充或扫描未展开目录树。
- 合并互相包含的监控根目录，避免同一子树建立重复递归 watcher；候选路径在全局稳定检测调度器中按规范化路径去重。
- 在进入稳定检测前按归档后缀和当前日志后缀配置预筛选，使用固定数量 worker、有界原始事件队列和有界候选集合处理事件风暴。
- watcher 溢出或事件不完整时仅恢复已知候选路径或受影响的直接父目录，禁止递归重扫整个监控根目录。
- 点击深层目录的新日志通知时，按路径逐层懒加载并展开祖先目录，选中目标节点并滚动到可见区域；失效路径从通知列表移除。
- 深层新日志尚未加载到目录树时，折叠的监控根和已加载祖先仍根据通知绝对路径显示未读高亮，不依赖 children 已存在。
- 增加深层下载、慢速写入、同名覆盖、重叠根目录、批量事件和空闲无扫描测试，并记录 Windows/macOS 原生 watcher 验证。

## Impact

- Affected specs: `directory-monitoring`、`log-notification`
- Affected code: `src-tauri/src/watcher.rs`、`src-tauri/src/lib.rs`、`src/App.tsx`、`src/components/DirTree.tsx`、目录树路径工具
- Runtime: 空闲时每个非重叠监控根保留一个递归 OS watcher，不轮询、不枚举未展开子树；负载与实际文件事件量而非磁盘文件总数相关
- Compatibility: 不改变现有监控目录、后缀筛选和目录树展开状态的持久化格式
