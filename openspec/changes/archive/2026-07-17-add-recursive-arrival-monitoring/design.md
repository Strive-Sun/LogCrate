## Context

`WatchState::start_watch` 当前同时承担目录结构同步和新文件稳定检测，并始终使用 `RecursiveMode::NonRecursive`。根目录启动时只监听直接子项；子目录只有在前端展开时才增加 watcher，折叠后立即释放。因此目录树的懒加载性能良好，但未展开子树中的日志文件完全不会进入通知链路。

监控整个盘符时不能通过递归扫描补偿：盘内可能包含数百万文件、权限受限目录和持续变化的构建缓存。可接受的方案必须只订阅原生事件，且内存、线程和补偿 IO 都有明确上限。

## Goals / Non-Goals

- Goals:
  - 监控根目录任意深度的新 zip 和受支持日志文件都能进入稳定检测并触发通知。
  - 未展开子目录继续保持不扫描、不加载到目录树。
  - 空闲开销不随根目录现有文件数量增长。
  - 高频写入时事件接收不被稳定检测阻塞，队列和 worker 数量保持有界。
  - Windows 与 macOS 使用相同的候选路径和去重语义。
- Non-Goals:
  - 不把未展开子树实时同步到左侧目录树。
  - 不保证第三方 watcher 在操作系统原生缓冲区溢出后绝对不丢事件。
  - 不在本 change 中增加排除目录 UI、通配规则或单独的性能设置。
  - 不通过定时全盘扫描实现递归发现。

## Decisions

### 分离递归到达监听与目录结构监听

- 每个有效监控根目录创建一个 `RecursiveMode::Recursive` 到达 watcher，只提取 create、content modify、rename-to 等可能代表文件到达的路径。
- 目录树继续使用现有的 `RecursiveMode::NonRecursive` 结构 watcher：根目录常驻，子目录随展开创建并随折叠释放。
- 结构 watcher 不再负责稳定检测，避免根目录文件同时被递归与非递归 watcher 重复调度；所有到达候选统一进入全局稳定检测调度器。
- 递归到达 watcher 不生成 `directory-changed`，因此深层事件不会突破前端懒加载边界。

### 合并重叠根与规范化路径

- 根据规范化后的绝对路径计算最小覆盖根集合：如果 `D:\` 已监控，则 `D:\logs` 不再创建第二个递归到达 watcher。
- 添加、移除或重命名监控根后重算覆盖集合；移除父根时为仍配置的子根补建 watcher。
- 稳定检测以规范化绝对路径为 key，并记录匹配的最具体配置根作为通知来源，避免重叠配置产生重复提示。
- Windows 路径比较遵循平台大小写语义；macOS 继续兼容 FSEvents 的规范化路径差异。

### 事件预筛选与有界稳定检测

- 进入稳定检测前忽略目录、访问事件和明显无关文件。当“显示全部”关闭时，仅保留已知归档后缀或当前配置的日志后缀；开启时允许所有普通文件进入后续分类。
- 临时下载名不会被提前通知；文件最终重命名为 `.zip`、`.log` 等目标名称时，rename-to 事件会重新进入候选流程。
- 使用一个应用级调度器，按路径合并重复 modify 并以 generation 使旧任务失效；固定少量 worker 并发轮询文件大小，worker 等待期间不阻塞 watcher 事件线程。
- 原始事件通道、待稳定路径集合和溢出补偿集合均使用常量容量；同一路径只占一个候选槽位，内存复杂度为 `O(根目录数 + 已展开目录数 + 容量上限)`，不随磁盘文件总数增长。

### 溢出处理不递归重扫

- 原始事件通道已满时，从被拒绝事件中提取路径放入有界、去重的补偿集合，由消费线程后续处理。
- 原生事件携带 `need_rescan`、rename 信息不完整或路径已变化时，只重查事件涉及的直接父目录或已知候选，不递归枚举监控根。
- 如果操作系统只报告无路径的全局溢出，记录降级诊断但不启动全盘扫描；这是有界资源与绝对不漏报之间的明确取舍。
- 目录树结构 watcher 仍可对其负责的单个已加载目录执行现有直接子项 rescan，不扩大到未展开子树。

### 点击通知后按路径定位

- 前端从通知携带的绝对路径中选择包含目标的最具体监控根，并计算“监控根 → 目标父目录”的祖先链。
- 点击通知后按祖先顺序调用现有 `expand_directory`，每次只读取该层直接子项并把 rescan 结果应用到目录树；不得为定位目标递归扫描兄弟子树。
- 全部祖先加载完成后设置目标选中状态，目录节点根据 reveal path 自动展开，目标行挂载后使用 `scrollIntoView({ block: 'nearest' })` 滚动到左栏可见区域。
- reveal path 仅用于本次定位，目标滚动完成后立即清理；不得让旧 reveal 状态在用户后续折叠/展开时绕过 `expand_directory` 自动打开已释放 watcher 的目录。
- 目录未读高亮直接判断未读通知绝对路径是否位于该目录之下，不依赖目标已经出现在 `children`；该纯字符串路径判断不访问磁盘。
- 如果某层读取失败或最终快照中不存在目标，视为通知已失效：移除通知并向用户提示，不保留无法定位的选中状态。
- Windows 路径匹配忽略盘符和路径组件大小写，同时兼容 `/` 与 `\`；macOS 保持大小写并复用 watcher 已规范化的路径。

## Performance Evaluation

### 空闲状态

- Windows 的 `ReadDirectoryChangesW` 和 macOS 的 FSEvents 由 `notify` 封装为递归原生订阅，不需要启动时递归 `read_dir`，也不需要周期轮询。
- 每个非重叠根增加一个 OS watcher；重叠根合并后，监控整个 `D:\` 通常只增加一个递归订阅。
- Rust 侧只保留固定容量通道、路径去重表和少量休眠 worker；空闲 CPU/IO 应接近零，内存不取决于盘内已有文件数。

### 活跃状态

- 事件接收成本近似 `O(E)`，其中 `E` 是实际文件系统事件数。监控整盘时，编译、解压、包管理器缓存等会显著增加 `E`，这是主要性能风险。
- 后缀预筛会在默认配置下排除大部分中间产物；同一路径的连续写入只更新 generation，不创建无限稳定检测任务。
- 稳定检测会对每个候选执行有限次数 metadata 查询；固定 worker 限制并发 IO，但极端批量下载的通知可能排队延迟。
- 禁止溢出时全盘重扫，避免一次事件风暴演变成遍历数百万文件的 IO 峰值。

### 验收基线

- 启动和空闲测试证明未对未展开子树调用递归扫描，等待期间不产生周期性目录 IO。
- 合成至少 10,000 条重复/混合事件，验证内存结构不超过配置容量、事件线程不等待稳定检测，并最终处理保留下来的归档候选。
- 在临时多层目录内慢速写入并最终重命名 zip，验证只产生一次通知。
- Windows 实机验证盘符级监听；macOS CI/实机验证递归事件、路径规范化和测试通过。

## Risks / Trade-offs

- 整盘高频变化会增加 CPU 唤醒和路径处理 → 后缀预筛、批处理、路径去重和固定 worker 限制成本。
- 系统原生事件缓冲区仍可能溢出 → 保留已知路径补偿并记录无路径降级，不以不可控全盘扫描换取表面完整性。
- 重叠根合并增加注册生命周期复杂度 → 用纯函数计算覆盖集合，并覆盖父根移除、子根接管测试。
- 不同平台 rename 事件形态不同 → 标准化 create/modify/rename-to 为同一候选动作，以最终路径存在性和稳定检测结果为准。
- 默认后缀预筛可能跳过无后缀文本 → 当前“显示全部”关闭时这类文件本就不会通知；开启“显示全部”后仍进入分类。
- 点击深层通知需要按目录深度执行多次 IPC/read_dir → 仅在用户点击时发生，复杂度为 `O(路径深度)`，不访问无关兄弟子树。

## Migration Plan

无需配置迁移。升级后从现有监控根目录计算递归覆盖集合并启动到达 watcher；回滚后恢复为只通知根目录和已展开目录的直接文件，不影响持久化配置。
