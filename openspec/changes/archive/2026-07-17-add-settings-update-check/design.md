## Context

LogPeek 的版本唯一来源是 `src-tauri/Cargo.toml`，Tauri 构建时会把该版本写入应用元数据。正式版本通过 `vX.Y.Z` tag 发布到 GitHub Releases。当前设置按钮没有事件绑定，Release 也只包含普通安装包，尚未生成 Tauri updater 所需的签名更新包与 `latest.json`。

## Goals / Non-Goals

- Goals:
  - 在应用内展示由 Tauri 元数据读取的真实版本号。
  - 支持启动自动检查和面板内手动检查，并复用同一检查逻辑。
  - 自动检查不打扰已是最新版的用户，手动检查始终给出结果。
  - 由用户确认下载后显示实时进度，完成签名验证后自动安装并重启。
  - 允许跳过指定版本，未来更高版本仍可再次提示。
  - 网络失败不影响日志查看和应用启动。
- Non-Goals:
  - 本次不检查草稿版、预发布版或 nightly 构建。
  - 本次不增加后台定时轮询；每次应用进程启动最多自动检查一次。
  - 本次不支持降级、更新通道切换或后台无提示安装。

## Decisions

### 设置面板与状态所有权

- 设置按钮打开顶栏右侧的轻量弹层，使用现有弹层与 backdrop 交互模式。
- `App` 持有自动检查开关和更新检查结果，设置面板只负责展示与触发操作，避免自动检查与手动检查形成两套状态。
- 自动检查开关默认开启，保存在 `localStorage` 的稳定 key 中；读取异常时回退为开启。

### 版本来源与官方 updater

- 桌面环境通过 `@tauri-apps/api/app` 的 `getVersion()` 读取当前构建版本，不复制 Cargo 版本到前端文件。
- 浏览器 mock 提供固定的开发版本，以便独立预览设置面板。
- 桌面环境使用 Tauri 2 官方 updater 插件读取固定 GitHub Releases `latest.json` endpoint，由插件完成版本判断、更新包下载和签名验证。
- 使用 process 插件在安装成功后重启应用；浏览器 mock 只模拟状态流，不执行真实安装。
- `tauri.conf.json` 开启 updater artifacts 并配置 endpoint 与签名公钥；Release workflow 通过 secrets 注入签名私钥和密码，生成清单、`.sig` 及 Windows/macOS 更新包。

### 提示策略

- 自动检查：启动后异步执行；发现更高版本时弹出一次更新提示，已是最新版或请求失败时不弹窗打扰用户。
- 手动检查：按钮进入 loading 状态并防止重复请求；完成后在设置面板内明确显示最新版、新版本或失败信息。
- 新版本提示展示当前版本与最新版本，并提供“跳过此版本”和“下载更新”两个操作。
- 跳过版本写入本地持久化设置；后续自动检查不再提示相同版本，但手动检查仍显示该版本并允许下载，且更高版本不受影响。

### 下载、安装与进度

- 用户选择“下载更新”后关闭跳过路径并进入单一更新任务，检查、下载或安装期间禁止重复触发。
- 使用 updater 下载事件累计已接收字节；已知总大小时显示百分比，未知总大小时显示不确定进度，下载完成事件将进度置为 100%。
- 下载完成后立即由 updater 校验签名并安装，不再要求第二次确认；安装成功后调用 relaunch 载入新版本。
- 下载、签名验证或安装失败时保留当前版本可用，展示失败阶段与重试操作，不把进度伪装为成功。

### 错误与安全边界

- 只访问固定的 `Strive-Sun/LogPeek` GitHub Releases updater endpoint，不接受用户输入的任意 URL。
- updater 必须验证由应用内置公钥对应私钥产生的签名；任何缺失或无效签名都拒绝安装。
- 签名私钥只存放在 GitHub Actions secrets，仓库、构建日志和应用包中只允许出现公钥。
- 更新元数据或网络错误归一化为用户可理解的失败状态，不阻断启动。

## Risks / Trade-offs

- GitHub 不可访问、限流或离线时检查/下载会失败 → 自动检查静默降级，用户操作显示可重试状态，当前版本不受影响。
- 更新包签名使发布流程依赖长期保存的私钥 → 私钥仅进入受保护 secrets，并在发布前增加必需配置检查。
- Windows 安装可能触发系统权限提示，macOS 更新受签名/公证状态影响 → 保留系统安全提示，不尝试绕过；分别验证两平台真实安装链路。
- 更新安装完成后必须重启，可能中断正在查看的会话 → 下载前明确告知将自动安装并重启，由用户主动确认开始。

## Migration Plan

无需业务数据迁移。首次运行新版本时自动检查开关按默认值开启；用户关闭或跳过版本后由本地设置持续保留。启用自更新前先生成 updater 密钥对，将公钥提交到 Tauri 配置，并由仓库管理员把私钥与密码写入 GitHub Actions secrets；旧版安装包继续可用，但只有包含 updater 插件与公钥的新版本能执行后续原地更新。
