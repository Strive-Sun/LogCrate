## ADDED Requirements
### Requirement: 日志查看测试覆盖
日志查看能力 SHALL 具备自动化单元测试,覆盖行偏移索引、边界行读取、行分隔符切分、编码解码、超长行截断与会话缓存清理,以在重构时守护正确性。

#### Scenario: 行索引与边界读取
- **WHEN** 运行行偏移索引的单元测试,含空行、末行无换行以及越界的 start/count
- **THEN** 测试断言索引与读取结果正确、越界被安全处理

#### Scenario: LF/CRLF 切分
- **WHEN** 运行行分隔符的单元测试,分别使用 LF 与 CRLF 输入
- **THEN** 测试断言按行正确切分且返回行不含行尾 `\r`/`\n`

#### Scenario: 编码解码
- **WHEN** 运行编码解码的单元测试,输入 UTF-8、GBK 与含 BOM 的 UTF-16
- **THEN** 测试断言均正确解码为可读文本

#### Scenario: 超长行截断
- **WHEN** 运行超长行的单元测试,输入超过阈值(如 64KB)的单行
- **THEN** 测试断言该行被截断并标记

#### Scenario: 会话缓存清理
- **WHEN** 运行会话生命周期的单元测试,关闭会话或触发 LRU 回收
- **THEN** 测试断言行索引与临时缓存文件被释放/删除
