## ADDED Requirements
### Requirement: 目录监控测试覆盖
目录监控能力 SHALL 具备自动化单元测试,覆盖类型判定、后缀筛选、配置持久化读写与失效目录处理,以在重构时守护正确性。

#### Scenario: 类型判定
- **WHEN** 运行 `classify` 的单元测试,分别输入 zip、裸文本与非文本文件
- **THEN** 测试断言分别被判定为 archive、file(日志)与被忽略/非日志

#### Scenario: 后缀筛选
- **WHEN** 运行后缀筛选的单元测试
- **THEN** 测试断言匹配后缀的文件被保留、不匹配的被过滤

#### Scenario: 配置持久化回环
- **WHEN** 运行配置读写的单元测试,写入目录列表与筛选规则后再读取
- **THEN** 测试断言读回的配置与写入一致

#### Scenario: 失效目录跳过
- **WHEN** 运行针对不存在目录的单元测试
- **THEN** 测试断言处理被跳过且不 panic
